Contexte du projet
Dans le cadre de votre formation à YouCode, vous êtes chargé de réaliser un projet fil rouge, qui constitue le projet de fin de formation. Ce projet a pour objectif de mobiliser et de valoriser l’ensemble des compétences acquises durant les deux années du cursus, tout en démontrant votre capacité à concevoir et à déployer une solution complète en intelligence artificielle.

Il s’agit d’un projet ambitieux et structurant, couvrant l’ensemble des compétences du référentiel, notamment :

Entraînement et optimisation de modèles de Machine Learning et de Deep Learning
Intégration de services d’IA externes et d’APIs
Conception et implémentation d’applications basées sur des LLM et du RAG
Automatisation et orchestration des pipelines de données
Mise en production, supervision et bonnes pratiques MLOps
Sécurité, gouvernance et conformité, incluant notamment JWT, RGPD et AI Act
L’objectif final est de livrer une application d’intelligence artificielle fonctionnelle, sécurisée, performante et déployée, répondant à un besoin métier réel et démontrant votre maîtrise des technologies et méthodologies enseignées.

Étapes du projet
1- Analyse des Exigences

Description :
Recueillir les besoins du client, rédiger les spécifications fonctionnelles et techniques, définir les contraintes, risques et les user stories pour guider le développement.

Livrables :
Cahier des charges / Spécifications fonctionnelles.
User stories + critères d’acceptation.
Documentation des exigences techniques.
2- Organisation & Modélisation

Description :
Structurer le projet, définir l’architecture globale, organiser les tâches et modéliser le système. Cela inclut la conception UML (cas d’utilisation, classes, séquences) et la modélisation des données (MCD/MPD).

Livrables :
Tableau de gestion de projet (Trello, Jira).
Diagrammes UML (use case, classes…).
MCD / MPD du système.
3- Sources de Données

Description :
Identifier, collecter et préparer les données provenant d’API, bases SQL, fichiers ou systèmes Big Data. Nettoyage, transformation, normalisation et agrégation pour produire un dataset exploitable.

Livrables :
Scripts d’extraction (API, scraping, SQL, Big Data).
Script ETL de nettoyage et préparation.
Dataset final prêt à l’usage.
Décrire le format, la qualité et les contraintes légales (RGPD/IA Act).
4- Pipelines & Data Engineering
Description :
Créer des pipelines automatisés (ETL/ELT) pour orchestrer la collecte, transformation et préparation des données. Utiliser des outils comme Airflow, Spark ou des scripts programmés.
Livrables :
Pipeline automatisé (Airflow, Spark…).
Journal/logs d’exécution des pipelines.
5- Développement IA

Description :
Concevoir, entraîner et optimiser un modèle IA/ML (classification, NLP, vision, prédiction…). Effectuer la validation, l’évaluation des métriques et le monitorage du modèle.

Livrables :
Modèle IA entraîné + fichier du modèle (pickle, joblib…).
Pipeline complète d'entraînement.
Rapport de performance (métriques, évaluation).
6- Intégration de Services Externes

Description :
Identifier et intégrer des services IA ou API externes (Azure, AWS, GCP, HuggingFace, OpenAI). Configurer les clés API, endpoints et faire un benchmark pour recommander la meilleure solution.

Livrables :
Benchmark comparatif des services.
Script d’intégration du service externe.
Documentation de configuration (clés, endpoints…).
7- Développement de l’API

Description :
Construire une API REST qui expose les données ou le modèle IA. Gérer les endpoints, la sécurité, la documentation (Swagger), les tests et la mise en production.

Livrables :
API REST fonctionnelle avec fastAPI.
Authentification JWT/OAuth2.
Documentation Swagger / OpenAPI.
8- Interface & Visualisation

Description :
Développer une interface utilisateur ou dashboard pour afficher les données, visualiser les résultats ou interagir avec l’API IA. Respect des normes UX/UI et accessibilité.

Livrables :
Application front-end (React).
Dashboard de visualisation (Grafana, Streamlit…).
Documentation d’utilisation.
9- Déploiement & MLOps

Description :
Mettre en place une chaîne CI/CD pour automatiser les tests, entraînement, packaging et déploiement du modèle IA. Intégrer monitoring, logs, alertes et gestion du drift.

Livrables :
Pipeline CI/CD (GitHub Actions, GitLab CI…).
Image Docker + configuration de déploiement.
Surveiller dérive et performance (Prometheus, MLflow, Evidently-AI).
10- Tests & Qualité

Description :
Mettre en place tous les tests nécessaires : unitaires, E2E, tests du modèle, tests API, tests d’intégration. Garantir la qualité du code via CI et mesures de couverture.

Livrables :
Dossier de tests automatisés.
Rapport de qualité et couverture.
Pipeline CI exécutant les tests.
11- Sécurité & Conformité

Description :
Appliquer les bonnes pratiques de sécurité (authentification, autorisation, cryptage, OWASP), assurer la conformité RGPD et gérer les logs et audit trails.

Livrables :
Mécanismes de sécurité configurés (JWT, OAuth…).
Politique RGPD / anonymisation des données.
Système de logs + documentation de conformité.
12- Rapport et présentation

Description :
Un Rapport détaillé qui décrit toutes les étapes de votre projet avec un fichier Trello ou Jira vous devez créer une présentation PPT / Canva / Prezzi ou autre outil qui résume et présente votre travail devant le jury de la soutenance.

Livrables :
Rapport détaillé décrivant chaque étape du projet.
Présentation (PPT, Canva, Prezzi) pour le jury.
Export des tâches depuis Trello ou Jira pour démontrer la gestion du projet.

Modalités d'évaluation
Une durée de 45 min organisée comme suit :

• Démo (≈ 15 min) : défense publique, présentation du contenu et des fonctionnalités clés.

• Code Review / Questions IA (≈ 15 min) : implémentations & architecture, modèles IA, APIs & services externes, MLOps & déploiement prod.

• Mise en situation (≈ 15 min) : debugging d’un modèle en prod avec monitoring, optimisation perf & montée en charge, analyse d’incident & plan de mitigation.
Livrables
• Cahier des charges : spécifications & architecture (diagrammes simples).
• Lien GitHub : repo structuré (code source, API, tests, scripts de déploiement, README exécutable).
• Rapport du projet : analyse technique, comparaison de modèles, métriques & recommandations.
• Support de présentation : slides + démonstration live de l’application déployée (résultats mesurables).
